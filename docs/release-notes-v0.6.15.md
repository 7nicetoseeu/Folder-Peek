# Folder Peek v0.6.15

这次更新重点放在“钉住窗口”的实际工作流优化上。

现在，Folder Peek 支持三态图钉、多个钉住窗口并存，以及脱离原级联后的独立拖动。对于需要同时摊开多个深层文件夹、来回取用文件的场景，这一版会顺手很多。

## 本次更新

- 新增三态图钉
  - 默认
  - 只钉在桌面
  - 全局置顶
- 图钉交互改为单击循环切换三种状态
- 支持同时保留多个已钉住窗口
- 钉住后的窗口不再因为打开其他文件夹、打开文件、点击外部或按 `Esc` 而一起消失
- 独立钉住窗口现在支持从标题区自由拖动
- 优化钉住窗口的父链/子链清理逻辑，只保留当前真正需要留下的窗口
- 修复多级展开后，从上层窗口重新使用 `Space + 左键拖动` 可能失灵的问题

## 细节说明

### 三态图钉

图钉按钮仍然只出现在当前最外层窗口。

每次单击按下面顺序循环：

`默认 -> 只钉在桌面 -> 全局置顶 -> 默认`

其中：

- `只钉在桌面`：窗口会保留，但不会继续压在后续打开的普通程序窗口之上
- `全局置顶`：窗口会继续保持在后续打开的程序窗口之上

### 多个钉住窗口并存

现在可以把不同展开链路中的最外层窗口依次钉住，并同时保留下来使用。

这意味着你可以同时摊开多个深层目录，不必反复重新展开。

### 独立拖动

当某个钉住窗口已经脱离原来的父级链路后，它会变成独立保留窗。

这时可以直接从标题区拖动它的位置；列表区域不会误触发整窗拖动。

## 适合谁更新

如果你平时会：

- 同时打开多个层级很深的文件夹
- 需要把几个常用目录短时间固定在桌面上
- 在多窗口之间来回取文件

这一版会比之前稳定很多，也更接近“摊开就能用”的状态。

## 下载与使用

- `win-x64`
- framework-dependent
- 运行前需要已安装 **.NET 8 Desktop Runtime**

运行后的设置、主题和索引缓存会写入：

`%LocalAppData%\FolderPeek`

## 已知说明

- 当前主手势仍然是 `Space + 左键拖动`
- 图钉按钮仍然只提供给当前最外层窗口，不会出现在中间层窗口
